So sánh Stream Processing

Đánh giá Kafka Streams vs Spark Streaming vs Apache Flink cho bài toán Real-Time GPS Tracking

Kafka Streams

Embedded Library (Java)

ĐÃ CHỌN

Ưu điểm chính

  • Sub-ms latency - Event-at-a-time
  • RocksDB embedded - No external state
  • Exactly-Once V2 - Native Kafka transactions
  • 1 file JAR - Deployment đơn giản

Mô hình xử lý

Event-at-a-time

• Process mỗi GPS point ngay khi đến

• Không đợi batch, latency sub-ms

Phù hợp cho demo

Deployment đơn giản, không cần cluster riêng

Spark Streaming

Micro-batching (Scala/Python)

Hạn chế chính

  • Batch interval - 0.5-2s minimum latency
  • GPS tracker bị giật - Không mượt mà
  • External state - Cần Redis/HDFS
  • Cluster riêng - Master/Worker architecture

Mô hình xử lý

Micro-batching

• Gom events thành batch nhỏ (0.5-2s)

• Process cả batch cùng lúc

Không phù hợp

Batch interval violates <500ms latency requirement

Apache Flink

Distributed Engine (Java/Scala)

Ưu điểm

  • Event-at-a-time - Latency rất thấp
  • State management - Built-in state backend
  • Exactly-once - Guaranteed processing

Nhược điểm

  • Cluster riêng - JobManager/TaskManager
  • Phức tạp vận hành - Setup, monitoring
  • Overhead - Cho dự án demo quy mô nhỏ

Production quy mô lớn

Performance tương đương Kafka Streams nhưng phức tạp hơn
