Data Model Design

Partition Key & Clustering Key - Tối ưu hóa truy vấn theo use case

trip_locations

GPS Time-Series Data

PRIMARY KEY

(trip_id, timestamp)

Partition Key: trip_id

• Tất cả GPS points của cùng chuyến đi trên cùng node

• Truy vấn 1 node duy nhất

Clustering Key: timestamp DESC

• Điểm mới nhất ở đầu partition

• LIMIT 1 trả về vị trí mới nhất

SELECT * FROM trip_locations
WHERE trip_id = ?
ORDER BY timestamp ASC;

driver_analytics

Weekly Performance Metrics

PRIMARY KEY

(driver_id, week_start_date)

Partition Key: driver_id

• Toàn bộ dữ liệu tài xế trên 1 partition

• Phân tích hiệu suất theo tuần

Clustering Key: week_start_date DESC

• Tuần mới nhất trước

• Pattern: "xem tuần gần đây"

SELECT * FROM driver_analytics
WHERE driver_id = ?
AND week_start_date >= '2024-01-01';

alerts

Speeding & Proximity Alerts

PRIMARY KEY

(driver_id, timestamp, alert_id)

Partition Key: driver_id

• Mỗi driver có 1 partition riêng

• Truy vấn cảnh báo theo tài xế

Clustering Key: timestamp DESC

• Alert mới nhất luôn ở đầu

• alert_id đảm bảo uniqueness

SELECT * FROM alerts
WHERE driver_id = ?
AND timestamp >= ?;
