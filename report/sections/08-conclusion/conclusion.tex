\section{Kết luận \& Hướng phát triển tương lai}

% --- 8.1 Conclusion ---
\subsection{Kết luận}
\label{sec:summary}

Dự án này đã thành công xây dựng một \textbf{Hệ thống theo dõi giao hàng thời gian thực} hoàn chỉnh, đáp ứng toàn bộ 8 yêu cầu nghiệp vụ (2 × n = 8, với n = 4 thành viên), bao gồm:

\bigskip
\begin{itemize}
    \item \textbf{4 yêu cầu xử lý thời gian thực (Kafka Streams):} Theo dõi vị trí, tính ETA động, giám sát tốc độ, và geofencing alerts. Kafka Streams đã chứng minh khả năng xử lý microsecond-level latency với stateful processing và stream-table joins — những khái niệm cốt lõi của stream processing trong bối cảnh Big Data.

    \item \textbf{4 yêu cầu quản lý dữ liệu (Cassandra):} Phát lại hành trình, phân tích hiệu suất tài xế, tính cước tự động, và bản đồ nhiệt. Cassandra đã chứng minh khả năng write-throughput cao (phù hợp với 17 GB GPS data/ngày) và schema design tối ưu cho time-series queries.

    \item \textbf{Ứng dụng hoàn chỉnh:} Customer, Driver, và Admin interfaces được kết nối real-time qua WebSocket, với dữ liệu được truyền từ GPX → Kafka → Kafka Streams → Cassandra → REST/WebSocket → React Map.
\end{itemize}

\bigskip
\noindent
Điểm nổi bật của kiến trúc:
\begin{itemize}
    \item \textbf{Polyglot programming} — mỗi tầng sử dụng ngôn ngữ tối ưu cho use case: Golang (concurrency), Java (Kafka Streams), Cassandra (time-series), React (UI).
    \item \textbf{Event-driven architecture} — các thành phần giao tiếp qua Kafka và WebSocket, không couple chặt với nhau.
    \item \textbf{Đánh giá có hệ thống} — đo được latency E2E P99 < 500ms, 0 malformed messages, và data correctness 100\%.
\end{itemize}

% --- 8.2 Future Enhancements ---
\subsection{Hướng phát triển tương lai}
\label{sec:future}

\bigskip
\begin{itemize}
    \item \textbf{Machine Learning cho ETA Prediction (Bonus):}
    \begin{itemize}
        \item Tích hợp ML model (ví dụ: Random Forest, LSTM) để dự đoán ETA chính xác hơn dựa trên historical traffic patterns, thời tiết, và giờ cao điểm.
        \item Dữ liệu huấn luyện: historical GPS traces từ \texttt{trip\_locations} trong Cassandra.
        \item Serve model bằng Kafka Streams transformer hoặc gRPC microservice riêng.
    \end{itemize}

    \item \textbf{Kubernetes Deployment:}
    \begin{itemize}
        \item Triển khai hệ thống lên Kubernetes cluster (Minikube cho dev, EKS/GKE cho production).
        \item Kafka (Strimzi operator) + Cassandra (K8ssandra) cho auto-scaling và self-healing.
        \item CI/CD pipeline với GitHub Actions + Helm charts.
    \end{itemize}

    \item \textbf{Multi-City Expansion:}
    \begin{itemize}
        \item Mở rộng hệ thống hỗ trợ nhiều thành phố (HCM, Hanoi, Da Nang).
        \item Sử dụng Cassandra token-aware routing để phân phối data theo city.
        \item Kafka topic partitioned theo city\_id + driver\_id.
    \end{itemize}

    \item \textbf{Real-Time Dashboard Optimization:}
    \begin{itemize}
        \item Tích hợp Grafana + Prometheus để monitor Kafka consumer lag, Cassandra read/write latency, và JVM metrics.
        \item Trực quan hóa topology Kafka Streams qua Kafka Streams DSL visualization.
    \end{itemize}

    \item \textbf{Advanced Geospatial Queries:}
    \begin{itemize}
        \item Thay heatmap đơn giản bằng Apache Sedona (GeoSpark) hoặc Cassandra's built-ingeohash support để query spatial aggregates phức tạp hơn.
        \item Tính năng tìm tài xế gần nhất bằng spatial index.
    \end{itemize}
\end{itemize}
